# Pemetaan Business Model ke Website PRESENSIGO

Business Model Canvas digunakan sebagai dasar keputusan konten dan fungsi.
Canvas tidak ditampilkan mentah pada halaman publik.

| Elemen bisnis | Penerapan pada website |
| --- | --- |
| Segmen pelanggan | Perusahaan, sekolah, universitas, instansi pemerintah, organisasi, komunitas, dan event organizer |
| Proposisi nilai | Anti-titip absen, validasi GPS real-time, monitoring terpusat, laporan otomatis, dan administrasi lebih efisien |
| Kanal | Website multi-page, WhatsApp konsultasi, email, dan form kontak |
| Hubungan pelanggan | Konsultasi kebutuhan, pemilihan paket, implementasi terarah, dan dukungan aktivasi |
| Arus pendapatan | Basic sebagai sistem siap pakai, Standard sebagai integrasi website, Premium sebagai website resmi plus sistem, serta penyesuaian implementasi |
| Aktivitas kunci | Konfigurasi QR/GPS, onboarding admin, monitoring, pelaporan, dan dukungan |
| Sumber daya kunci | Platform PRESENSIGO, dashboard, scanner mobile, data kehadiran, dan tim implementasi |
| Mitra kunci | Penyedia infrastruktur, pihak institusi, dan mitra integrasi jika diperlukan |
| Struktur biaya | Pengembangan, hosting, dukungan, konfigurasi, pelatihan, dan pemeliharaan |

## Peran Setiap Halaman

- Home: membangun urgensi, menjelaskan solusi, memberi bukti visual, dan
  mengarahkan ke konsultasi.
- Tentang: menjelaskan prinsip produk dan alasan operasional.
- Layanan: membantu calon pelanggan memilih tingkat solusi.
- Galeri: menunjukkan bentuk dashboard dan proses tanpa klaim screenshot resmi.
- Testimoni: menyiapkan format social proof; konten saat ini adalah placeholder.
- FAQ: mengurangi hambatan sebelum konsultasi.
- Kontak: mengumpulkan konteks kebutuhan dan mengarahkan ke WhatsApp.

## Jalur Konversi

1. Loader singkat membangun konteks bahwa sistem sedang memvalidasi QR, GPS,
   dan dashboard tanpa memperlambat akses.
2. Pengunjung mengenali masalah pada hero dan bagian masalah.
3. Manfaat bisnis dan pinned workflow menjelaskan hasil serta alur produk.
4. Fitur, product showcase, dan paket menurunkan ketidakpastian.
5. FAQ menjawab hambatan umum.
6. CTA mengarahkan ke WhatsApp, email, atau form konsultasi.

## Model Implementasi

| Paket | Job to be done | Nilai bisnis |
| --- | --- | --- |
| Basic | Menjalankan absensi digital tanpa website baru | Aktivasi lebih cepat dan ruang lingkup sederhana |
| Standard | Menambahkan PRESENSIGO ke website existing | Pengalaman pengguna tetap menyatu dengan kanal digital organisasi |
| Premium | Membangun website resmi dan sistem absensi sekaligus | Satu implementasi untuk branding, informasi, konversi, dan operasional |

Standard diposisikan sebagai pilihan paling fleksibel bagi organisasi yang
sudah memiliki website. Premium memperluas value proposition ke pembuatan
website resmi, bukan hanya fitur absensi.

## Prinsip Copywriting

- Menjual hasil operasional, bukan daftar teknologi.
- Tidak membuat klaim klien, angka penghematan, atau atribusi yang belum sah.
- Menggunakan istilah yang mudah dipahami pengambil keputusan dan admin.
- Menempatkan QR, GPS, dashboard, dan laporan sebagai satu alur kerja.
